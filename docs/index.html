api

The top level 'api' is what is available to an app after the Citrix WebRTC SDK has been imported. It contains various top-level functions that pertain to Citrix WebRTC SDK global instance, as well as event emitter methods.

api
Types
▸ CitrixError

CitrixError

Type: Object

Properties
code (string) : One of api.errorCodes

▸ setup(SDK, timeout)

Initializes the Citrix UC SDK and sets up the WebRTC SDK to run in Citrix proxy mode.

setup(SDK: Object, timeout: number): Promise
Parameters
SDK (Object) The WebRTC JS SDK.
timeout (number = 10000) The time to wait, in milliseconds, for the Citrix SDK to initialize.
Returns
Promise: Resolves upon successful completion
Throws
  • api.CitrixError: Will throw an error with code INVALID_STATE if Citrix proxy mode is already setup.
  • api.CitrixError: Will throw an error with code INVALID_PARAM if provided SDK parameter is invalid.
  • api.CitrixError: Will throw an error with code INVALID_STATE if active calls are ongoing.
  • api.CitrixError: Will throw an error with code VERSION_MISMATCH if WebRTC JS SDK and Citrix WebRTC SDK versions mismatch.
  • api.CitrixError: Will throw an error with code CITRIX_SDK_ERROR if Citrix UC SDK fails to initialize.
Example
import citrixClient from '@rbbn/citrix-webrtc-sdk/citrix'

// Setup proxy mode on WebRTC SDK client by passing in the created client.
try {
  await citrixClient.setup(webrtcClient)
} catch (error) {
  // See Throws section and api.errorCodes documentation for list of possible errors.
}
▸ teardown()

Tears down the Citrix UC SDK and returns the WebRTC JS SDK to non-proxy mode.

This API will also trigger a device update within the WebRTC JS SDK after successfully tearing down the proxies.

teardown(): Promise
Returns
Promise: Resolves upon successful completion.
Throws
  • api.CitrixError: Will throw an error if active calls are ongoing.
Example
try {
  citrixClient.teardown()
} catch (error) {
  // See Throws section and api.errorCodes documentation for list of possible errors.
}
▸ isSetup()

Returns whether proxy mode has been setup or not.

isSetup(): boolean
Returns
boolean: Returns true if proxy mode is setup, false otherwise.
Example
log(citrixClient.isSetup())     // false
await citrixClient.setup(webrtcSDK)
log(citrixClient.isSetup())     // true
citrixClient.teardown()
log(citrixClient.isSetup())     // false

▸ on(type, listener)

on(type: string, listener: Function)

▸ subscribe(listener)

Adds a global event listener to the SDK instance. The listener is a function that will receive all events.

subscribe(listener: Function)
Parameters
listener (Function) The event listener function. The function will be called with parameters (type, ...args), where args depend on the event type.
Example
import citrixClient from '@rbbn/citrix-webrtc-sdk/citrix'

// Listen for all events emitted by the SDK.
function eventHandler(type, ...eventData) {
   // handle the event data
}
citrixClient.subscribe(eventHandler)
▸ unsubscribe(listener)

▸ adjustVideoPosition(xOffset, yOffset)

Sets an offset by which all video elements will be adjusted. Must be called prior to rendering the video stream to be adjusted.

adjustVideoPosition(xOffset: number, yOffset: number): Promise
Parameters
xOffset (number) The number of pixels to adjust video elements left or right. A negative offset moves elements left, a positive number moves them right.
yOffset (number) The number of pixels to adjust video elements up or down. A negative offset moves elements up, a positive number moves them down.
Returns
Promise: Resolves upon successful completion
Throws
  • api.CitrixError: Will throw an error with code INVALID_STATE if Citrix proxy mode has not been set up.
  • api.CitrixError: Will throw an error with code INVALID_STATE if Electron IPC functions have not been setup correctly.
  • api.CitrixError: Will throw an error with code INVALID_PARAM if the provided offset values are not of type 'number'.
Example
import citrixClient from '@rbbn/citrix-webrtc-sdk/citrix'

try {
  await citrixClient.setup(webrtcClient)

  // move all video elements right by 40 pixels, up by 200 pixels
  await citrixClient.adjustVideoPosition(40, -200)

} catch (error) {
  // See Throws section and api.errorCodes documentation for list of possible errors.
}

▸ errorCodes

Possible error codes for the errors thrown by the Citrix WebRTC SDK.

Each error thrown by the SDK is in the format of api.CitrixError. These error objects include a code and message. The codes provide a basic hint as to what the error is as per descriptions below. The message provides more context about the error encountered.

errorCodes

Type: Object

Properties
CITRIX_SDK_ERROR (string) : A CITRIX_SDK_ERROR error indicates that the Citrix UC SDK threw an error.
INVALID_PARAM (string) : An INVALID_PARAM error indicates that an invalid parameter was passed to the API.
INVALID_STATE (string) : An INVALID_STATE error indicates that SDK is in an invalid state to proceed with the operation.
VERSION_MISMATCH (string) : A VERSION_MISMATCH error indicates that the provided WebRTC SDK version does not match the Citrix WebRTC SDK version.
Example
import citrixClient from '@rbbn/citrix-webrtc-sdk/citrix'
try {
  await citrixClient.setup(rbbnWebRTC)
} catch (error) {
  // Error contains a `code` and `message` property.
  const { code, message } = error
  switch (code) {
    case citrixClient.errorCodes.CITRIX_SDK_ERROR:
      // Handle error originating from the Citrix UC SDK.
    case citrixClient.errorCodes.INVALID_PARAM:
      // Handle the error with the parameter provided.
    case citrixClient.errorCodes.INVALID_STATE:
      // Handle an SDK not being in the correct state for the API call.
    case citrixClient.errorCodes.VERSION_MISMATCH:
      // Handle the two SDK's not being on the same version.
  }
}
▸ eventTypes

Possible events emitted by the Citrix WebRTC SDK

eventTypes

Type: Object

Properties
CITRIX_CONNECTED (string) : Citrix Webrtc redirection initiated successfully
CITRIX_DISCONNECTED (string) : Citrix Webrtc redirection failed to start successfully or failed after connection
CITRIX_E911_DATA_CHANGED (string) : Client E911 data has changed; updated data should be retrieved via api.getE911Data
Example
import citrixClient from '@rbbn/citrix-webrtc-sdk/citrix'
// Start listening for events
citrixClient.on(citrixClient.eventTypes.CITRIX_E911_DATA_CHANGED, data => {
  // Do something with data
})
